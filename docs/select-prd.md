# Select 组件 — 产品需求文档（PRD）

**组件**：表单场景单选下拉 **Select**  
**适用范围**：`@component-ai/react-ui`、`@component-ai/vue-ui`  
**关联文档**：[交互稿](./select-interaction-spec.md) · [开发者 API](./select-api.md)

---

## 1. 背景与定位

### 1.1 要解决什么问题

业务表单需要「从一组选项中选 **一个** 值」的控件；在长列表或后端分页场景下，用户需要通过 **搜索** 缩小范围或触发远程查询。

### 1.2 产品定义

本组件是 **Select（选值）**，不是：

- 导航 / 表格「更多」类 **Menu（操作菜单）**；
- 自由拼装任意内容的 **Popover**；
- **Combobox** 的全套 APG 形态（见非目标）。

### 1.3 成功标准

| 编号 | 标准 |
|------|------|
| S1 | 用户能完成单选，且选中值可与外层表单状态同步（受控 / 非受控）。 |
| S2 | 默认提供搜索能力，支持本地过滤与远程按关键字加载。 |
| S3 | 异步加载可被取消，避免竞态与多余请求占资源。 |
| S4 | React 与 Vue 实现 **语义与行为一致**，便于统一 UX 与文档。 |

---

## 2. 范围：非目标（本期不做）

| 编号 | 说明 |
|------|------|
| O1 | 多选、标签输入、批量选择。 |
| O2 | 树形 / 级联 / 分组折叠等复杂结构（除非另立组件）。 |
| O3 | 选项行完全自定义渲染（本期以 **文本标签** 为主；空状态扩展能力见 API 文档）。 |
| O4 | 等同于 WAI-ARIA Combobox 模式的完整实现（本期采用「触发器 + 可选搜索 + listbox」的约定子集）。 |

---

## 3. 功能需求（验收口径）

| ID | 需求 | 验收要点 |
|----|------|----------|
| F1 | 面板展开 / 收起 | 点击触发器切换；点击组件外关闭；`Escape` 关闭（细则见交互稿）。 |
| F2 | 展示当前选中 | 触发器展示当前项文案（label）；无选中时展示占位文案（placeholder）。 |
| F3 | 选项列表 | 列表展示可选项；支持单项禁用；需区分「当前选中」与「键盘高亮」（交互稿）。 |
| F4 | 键盘可用 | 核心路径仅用键盘可选中一项（按键表见交互稿）。 |
| F5 | 搜索默认开启 | 产品默认 `filterable = true`；关闭时为纯列表无搜索框。 |
| F6 | 远程查询防抖 | 异步模式下关键字变更经防抖后再请求；空关键字首开不误等防抖（细则见 API）。 |
| F7 | 同步数据源 | 支持静态 `options`；可搜索时支持本地过滤规则（默认 + 可自定义）。 |
| F8 | 异步数据源 | 支持 `loadOptions`；打开面板与查询变化触发加载；取消策略见 API。 |
| F9 | 自定义业务对象 | 选项可为任意结构，通过映射函数产出 value / label / disabled（默认字段约定见 API）。 |
| F10 | 整块不可用 | `disabled` 时不可展开、不可改值。 |
| F11 | 空列表反馈 | 无数据或加载结束仍无数据时，有明确占位（渠道差异见 API）。 |

---

## 4. 数据源与一致性（产品规则）

为避免「静态列表 + 远程列表」混用含义不清：

| 模式 | 规则 |
|------|------|
| **仅同步** | 仅使用 `options`；列表完全来自传入数组。 |
| **仅异步** | 提供 `loadOptions` 时，**列表以异步返回为准**；不与 `options` 合并为同一列表。 |
| **失败 / 中止** | 异步失败或请求中止后，对用户表现为 **当前无可选项**（具体文案与加载态见交互稿 / API）。 |

---

## 5. 依赖与约束（产品侧）

- 样式依赖：使用者须在宿主应用中引入对应包的 **`style.css`**，否则视觉规格无法保证。
- 文案：`placeholder`、搜索框 `aria-label`、空状态等后续可做国际化替换（本期中文默认）。
