# Checkbox 组件 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `@component-ai/react-ui` 与 `@component-ai/vue-ui` 中交付 **完整、可独立使用** 的 `Checkbox` 原子组件（受控/非受控 `checked`、`indeterminate` 视觉态、`disabled`、表单 `name`/`value`、可访问的标签内容），供后续 VirtualGrid 行选中 / 全选接线；双端 API 语义对齐，命名跟从目标库习惯。

**Architecture:** `Checkbox` 是纯 UI 原子组件，**不依赖** `@component-ai/grid-core`——多选/全选/级联的状态机（`toggleKey`、`selectAllKeys`、`isAllSelected`、`isIndeterminate`）已在 grid-core 落地（见 `packages/grid-core/src/selection.ts`），职责边界见设计规格 §7：核心状态机在 core，UI 只渲染单个 checkbox 的 `checked` / `indeterminate` 视觉。VirtualGrid 接线阶段会用 grid-core 的选中函数算出每行/全选 checkbox 的 `checked` 与 `indeterminate` 值喂给本组件——**本计划不做该接线**。

视觉与交互采用原生 `<input type="checkbox">`：语义、键盘、表单集成免费获得；`indeterminate` 是 DOM 属性（非 HTML attribute），需要通过 ref 在挂载/更新时赋值。标签内容用原生 `<label>` 包裹 `<input>` + 内容（React 用 `children`，Vue 用默认 slot），点击标签文本即可切换，不需要手动 `id`/`htmlFor` 管理。

**Tech Stack:** TypeScript、React 19、Vue 3.5、TSX、Vite 8、Vitest、Testing Library / Vue Test Utils、Tailwind v4

**规格来源:** [`docs/superpowers/specs/2026-07-09-virtual-grid-migration-design.md`](../specs/2026-07-09-virtual-grid-migration-design.md) §5、§6（P1）、§7  
**路线图:** [`2026-07-09-virtual-grid-roadmap.md`](./2026-07-09-virtual-grid-roadmap.md)（顺序 3）  
**前置:** 无强制前置；grid-core 的 `selection.ts` 已存在（`5b828af`），但本计划不引用它。

## API 契约（双端对齐）

| 概念 | React | Vue |
|------|-------|-----|
| 选中值 | `checked` / `defaultChecked` / `onCheckedChange` | `modelValue` + `onUpdate:modelValue`（`v-model`），或 `defaultModelValue` 非受控 |
| 半选（视觉态，不代表值） | `indeterminate?: boolean`（默认 `false`） | 同 |
| 禁用 | `disabled?: boolean`（默认 `false`） | 同 |
| 必填（表单校验） | `required?: boolean`（默认 `false`） | 同 |
| 表单字段名/值 | `name?: string` / `value?: string` | 同 |
| 标签内容 | `children?: ReactNode`（渲染在 `<label>` 内，`<input>` 之后） | 默认 slot |
| 样式 | `className?: string` | `class?: string` |
| 变更事件 | `onCheckedChange?: (checked: boolean) => void` | `update:modelValue`（`(checked: boolean) => void`）；额外触发 `change` 事件（与 Select 现有约定一致） |

**行为**

- 受控判定：React 传入 `checked` 时以 props 为准；未传则用内部 `useState(defaultChecked)`。Vue 传入 `modelValue` 时以 props 为准；未传则用内部 `ref(defaultModelValue)`（对齐 `Select.tsx` 的 `modelValue` / `defaultModelValue` 模式）。
- `indeterminate` 为纯视觉态：为 `true` 时通过 DOM ref 设置 `input.indeterminate = true`，并把 `aria-checked` 覆盖为 `"mixed"`（辅助技术优先读 aria-checked）；`checked` 的实际值语义不受 `indeterminate` 影响，切换点击仍按当前 `checked` 值翻转。
- 点击 `<label>`（包括其内的文本内容）与点击 `<input>` 效果一致，原生 `<label>` 包裹行为免费获得。
- `disabled` 时 `<input disabled>`；鼠标样式与透明度体现在容器 `<label>` 上（`cursor-not-allowed` / `opacity-50`）。
- 无障碍：使用原生 `role="checkbox"`（由 `<input type="checkbox">` 隐式提供），无需手写 `role`；可通过 `getByRole("checkbox", { name })` 定位，`name` 来自 `<label>` 包裹的文本内容（accessible name 计算）。

- [x] **Step 1: 实现组件**

Create `packages/react-ui/src/components/Checkbox.tsx`:

```tsx
import {
  useEffect,
  useRef,
  useState,
  type ChangeEvent,
  type ReactNode,
} from "react";

export type CheckboxProps = {
  checked?: boolean;
  defaultChecked?: boolean;
  indeterminate?: boolean;
  disabled?: boolean;
  required?: boolean;
  name?: string;
  value?: string;
  className?: string;
  children?: ReactNode;
  onCheckedChange?: (checked: boolean) => void;
};

export function Checkbox({
  checked: checkedProp,
  defaultChecked = false,
  indeterminate = false,
  disabled = false,
  required = false,
  name,
  value,
  className = "",
  children,
  onCheckedChange,
}: CheckboxProps) {
  const [uncontrolled, setUncontrolled] = useState(defaultChecked);
  const checked = checkedProp ?? uncontrolled;
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    if (inputRef.current) inputRef.current.indeterminate = indeterminate;
  }, [indeterminate]);

  function handleChange(e: ChangeEvent<HTMLInputElement>) {
    const next = e.target.checked;
    if (checkedProp === undefined) setUncontrolled(next);
    onCheckedChange?.(next);
  }

  const wrapperCls =
    `inline-flex items-center gap-2 text-sm text-slate-800 ${
      disabled ? "cursor-not-allowed opacity-50" : "cursor-pointer"
    } ${className}`.trim();

  const boxCls =
    "h-4 w-4 shrink-0 rounded border border-slate-300 text-sky-600 accent-sky-600 focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-sky-600 disabled:cursor-not-allowed";

  return (
    <label className={wrapperCls}>
      <input
        ref={inputRef}
        type="checkbox"
        checked={checked}
        disabled={disabled}
        required={required}
        name={name}
        value={value}
        aria-checked={indeterminate ? "mixed" : checked}
        className={boxCls}
        onChange={handleChange}
      />
      {children}
    </label>
  );
}
```

注意：`checked` 传给原生 `<input>` 时必须始终提供 `onChange`（受控 input 规则），已通过 `handleChange` 满足；`aria-checked` 覆盖了原生隐式的 `aria-checked`，测试与 AT 均以此为准。

Create `packages/vue-ui/src/components/Checkbox.tsx`，行为与 React 对齐，`modelValue` / `defaultModelValue` 模式与 `Select.tsx` 一致：

```tsx
import {
  defineComponent,
  onMounted,
  ref,
  watch,
  type PropType,
} from "vue";

export const Checkbox = defineComponent({
  name: "Checkbox",
  props: {
    modelValue: { type: Boolean, default: undefined },
    defaultModelValue: { type: Boolean, default: false },
    indeterminate: { type: Boolean, default: false },
    disabled: { type: Boolean, default: false },
    required: { type: Boolean, default: false },
    name: { type: String, default: undefined },
    value: { type: String, default: undefined },
    class: { type: String, default: "" },
  },
  emits: {
    "update:modelValue": (_checked: boolean) => true,
    change: (_checked: boolean) => true,
  },
  setup(props, { emit, slots }) {
    const isControlled = () => props.modelValue !== undefined;
    const internal = ref(props.defaultModelValue);
    const inputRef = ref<HTMLInputElement | null>(null);

    function applyIndeterminate() {
      if (inputRef.value) inputRef.value.indeterminate = props.indeterminate;
    }
    onMounted(applyIndeterminate);
    watch(() => props.indeterminate, applyIndeterminate);

    function handleChange(e: Event) {
      const next = (e.target as HTMLInputElement).checked;
      if (!isControlled()) internal.value = next;
      emit("update:modelValue", next);
      emit("change", next);
    }

    return () => {
      const checked = isControlled() ? props.modelValue : internal.value;
      const wrapperCls =
        `inline-flex items-center gap-2 text-sm text-slate-800 ${
          props.disabled ? "cursor-not-allowed opacity-50" : "cursor-pointer"
        } ${props.class}`.trim();
      const boxCls =
        "h-4 w-4 shrink-0 rounded border border-slate-300 text-sky-600 accent-sky-600 focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-sky-600 disabled:cursor-not-allowed";

      return (
        <label class={wrapperCls}>
          <input
            ref={inputRef}
            type="checkbox"
            checked={checked}
            disabled={props.disabled}
            required={props.required}
            name={props.name}
            value={props.value}
            aria-checked={props.indeterminate ? "mixed" : checked}
            class={boxCls}
            onChange={handleChange}
          />
          {slots.default?.()}
        </label>
      );
    };
  },
});
```
